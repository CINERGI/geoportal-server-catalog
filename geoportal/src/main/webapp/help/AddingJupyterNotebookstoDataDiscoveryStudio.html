Adding Jupyter Notebooks to Data Discovery Studio: Guidelines

Data Discovery Studio (DDS) lets users explore geoscience resources with Jupyter notebooks. Using a document ID passed from DDS, such notebooks may retrieve additional resource metadata, download the referenced data or invoke services, generate visualizations and statistical summaries, or perform other tasks. Several sample notebooks are available on github at https://github.com/CINERGI/jupyter-cinergi. This guide will help you explore the content of sample notebooks, clone a default github repository into your account, add your own notebooks based on the samples, and call these notebooks from DDS to explore datasets of the type that you are interested in. For a brief introduction to Jupyter notebooks see https://www.dataquest.io/blog/jupyter-notebook-tutorial/

  1. Login to your github account (or create one), at https://github.com 
  2. Open https://github.com/CINERGI/jupyter-cinergi in your browser, and fork this repository into your account. The Fork button is at the top right. As a result, you’ll have a copy of the sample notebooks under your account. The address of your new repository will be https://github.com/<your_account_name>/jupyter-cinergi.
  3. There are two types of notebooks in the repository (not counting configuration and service files). These are “dispatchers” and “operations”. The dispatcher notebook may merely list available “operations” notebooks, or provide a smarter way to associate notebooks with types of resources. The CinergiDispatch.ipynb, in the root of your repository, is the main entry point, and it may reference other dispatchers. “Operations” notebooks reside in “operations/cinergi” and implement visualization, analysis, modeling and other functions. Both dispatchers and operations accept parameters from DDS (currently, a document ID for a selected resource.)
  4. Open the CinergiDispatch.ipynb notebook in your favorite editor and explore it. This notebook gets called when you click the “Studio” link in DDS search results.
  1. If you intend to modify or add notebooks, the best option is to download the repository to your local computer and explore it in your local Jupyter setup, then push your edits and added notebooks back to github. Here are the instructions.
  1. Explore the functions of each cell of the CinergiDispatch.ipynb notebook:
  1. On the first step, the notebook will accept document ID from DDS search results.
  2. On the second step, it will print out the accepted parameters; check if the document ID is passed correctly
  1. Step 2a: optionally, you can execute another version of the notebook dispatcher (you can skip it for now)
  2. Step 2b: if you want to switch to notebooks in your repository instead of the default one, you need to execute this step. It will clone your repository into the current container (into a “myclone” folder) and adjust the references. Skip this step if you want to run the default notebooks or if you are already running CinergiDispatch.ipynb in myclone.
  1. On the third step, you select an operation to run. When you create a new notebook for some type of DDS resource, you may want to add it to this menu. For example, you add a line like this below item #4 in the menu:

 '5. MyResource-explore': 'cinergi/MYR-explore.ipynb',
You then also need to place the notebook for your resource type in the “operations/cinergi” folder of your cloned repository. We’ll cover the structure of such “operation” notebooks in the next section.

  1. On the fourth step, the notebook will formulate the URL to a selected notebook. Click this URL to open it.

  1. Now, we’ll explore the content of a sample “operation” notebook, using operations/cinergi/ndbc-explore.ipynb as an example.
  1. The first cell of the notebook retrieves document ID from the URL - in exactly the same way as CinergiDispatch.ipynb did. You’ll need to copy this cell your new notebook
  2. The second cell imports packages used by this notebooks. Chances are that your new notebook will require other packages. In that case, install them directly from the notebook using !pip install. In more difficult cases, contact us
  3. The third cell retrieves metadata from DDS given the Document ID. In the NDBC case, this cell also presents a list of “distributions” (pointers to actual datasets or services referenced in the metadata.) Chances are, you will also need to show “distributions”, so that you can retrieve the datasets and do something with them in subsequent cells. The rest of the content is up to you.
